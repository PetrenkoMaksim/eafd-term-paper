MAIN IDEA OF THE PAPER / PROJECT

The EAFD pipeline does NOT train event-sequence embeddings itself. It expects precomputed embeddings as input.

There are two separate stages:

Stage A:
  raw event data -> PTLS preprocessing -> train CoLES/MeLES embeddings -> export data/embeddings into EAFD format

Stage B:
  original EAFD repo uses:
    transactions.csv
    target.csv
    test_ids.csv
    embeddings/coles.pickle
  then runs LLM-based feature generation with gpt-oss-20b or API and evaluates generated features with CatBoost.

Important distinction:
  - gpt-oss-20b / gpt-oss-120b is ONLY the LLM feature-generation agent.
  - It is NOT the event-sequence embedding model.
  - CoLES/MeLES embeddings are trained separately with PTLS.
